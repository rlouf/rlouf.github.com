Minimal coupling

I like to think of mcx as being the composition of three loosely coupled libraries rather than a full library:

  1. The core, which allows to build graphical models and compile them to a forward sampling function or a log-probability density function;
  2. The distributions which implement, well, distribution. Most of the heavy lifting will be transfered to JAX’s scipy and numpy in a near future. We just add utilities to check the parameters’ domain and ensure that value stay in the support;
  3. The inference engine, a collection of momentum generators/kinetic energies, integrators, adaptive schemes that can be combined into warmup and inference kernels.

[there should be no jitting in the core which should be JAX-agnostic]

With the root module as the glue between the 3.
